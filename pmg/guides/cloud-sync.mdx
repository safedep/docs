---
title: Cloud Sync
description: "Sync PMG activity to SafeDep Cloud for centralized monitoring and reporting"
---

This guide walks through enabling SafeDep Cloud sync in PMG so your package activity is visible in one place.

## Prerequisites

- Install PMG using the [quickstart guide](/pmg/quickstart)
- Have a SafeDep Cloud tenant ID and tenant domain (see [Cloud quickstart](/cloud/quickstart))

## Configure cloud sync

<Steps>
  <Step title="Run the PMG setup installer">
    ```console
    $ pmg setup install

    █▀█ █▀▄▀█ █▀▀  From SafeDep (github.com/safedep/pmg)
    █▀▀ █░▀░█ █▄█  version: v0.9.0 commit: c3a351

    ✓ PMG aliases installed successfully
      Installed to:  /home/kunalsingh/.pmg.rc
      Config at:     /home/kunalsingh/.config/safedep/pmg
      Restart your terminal or source your shell to use the new aliases
    ```
  </Step>

  <Step title="Enable cloud sync in the PMG config">
    Open the PMG config file at `~/.config/safedep/pmg/config.yml` (use `pmg setup info` if you need to confirm the path) and enable cloud sync while keeping any existing settings:
    ```yaml
    # ~/.config/safedep/pmg/config.yml
    cloud:
      enabled: true
    # other existing settings...
    ```
  </Step>

  <Step title="Login to SafeDep Cloud">
    Authenticate with your tenant details:
    ```bash
    pmg cloud login
    ```
    Enter your **tenant ID** and **tenant domain** when prompted.
  </Step>

  <Step title="Sync to SafeDep Cloud">
    ```bash
    pmg cloud sync
    ```
    Run this after login and whenever you want to push new local activity to SafeDep Cloud. Common options are after major dependency updates, after CI runs, or on a nightly schedule.
  </Step>
</Steps>
